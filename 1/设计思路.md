# 设计思路

## 项目定位

这组图片以原始照片 `1.jpg` 为基础，保留“隔着栏杆观看花田”的核心视觉关系，并将它扩展为更适合前端界面使用的电影感视觉资产。

整体方向不是单纯修图，而是把原图转化为一套可以用于网页 Hero、移动端启动页、登录页、作品集首页或自然影像类产品界面的背景系统。

## 核心视觉概念

原图的重点是“前景阻隔”和“远处自然景观”的对比：虚化的栏杆形成观看边界，黄色花田和天空提供自然、温暖、安静的情绪。

扩展时保留了这个叙事结构，并强化了以下特征：

- 电影感逆光：低角度夕阳制造更强的空间层次和情绪张力。
- Teal & Gold 调色：蓝绿色阴影搭配金色高光，让画面更接近电影剧照。
- 景深关系：栏杆保持虚化和压暗，花田与天空承担主要视觉信息。
- UI 留白：部分版本主动减少左侧或上方细节，方便叠加标题、按钮和导航。
- 边缘晕影：压暗画面边缘，提高界面文字的可读性。

## 文件说明

### `1.jpg`

原始参考图。

它提供了基本构图：栏杆、花田、天空、云层，以及一种安静但带有距离感的观看视角。

### `1-cinematic.png`

电影感增强版本。

这个版本保留原图构图，重点提升光影、色彩和氛围。适合用作视觉基准图，也可以作为文章封面、摄影专题图或情绪板素材。

### `2-web-hero-background.png`

网页 Hero 背景主版本。

画面左侧保留了较大面积的干净空间，适合放置首页标题、副标题和 CTA 按钮。右侧和下方保留花田与栏杆细节，能在不干扰文字的前提下维持视觉记忆点。

推荐使用场景：

- 官网首屏 Hero
- 摄影作品集首页
- 自然、旅行、影像类产品落地页
- 需要沉浸式背景的登录页

### `2-web-hero-background-alt.png`

网页 Hero 背景备选版本。

相比主版本，这张更明亮、更暖，夕阳氛围更强。适合需要更强情绪感染力的页面，但叠加文字时建议增加轻微暗色遮罩。

推荐遮罩示例：

```css
background:
  linear-gradient(90deg, rgba(8, 12, 14, 0.66), rgba(8, 12, 14, 0.18)),
  url("./2-web-hero-background-alt.png") center / cover no-repeat;
```

### `3-frontend-ui-mockup.png`

前端 UI 视觉概念图。

这张不是单纯背景，而是完整的界面方向参考。它展示了如何把电影感背景与现代 Web UI 结合：顶部导航、左侧 Hero 信息、右侧半透明信息面板、图表和内容预览。

适合作为：

- 设计提案图
- 前端页面临摹参考
- 作品集展示图
- UI 风格探索图

需要注意：这张图上已经包含界面元素，不建议直接作为网页背景使用。

### `4-mobile-ui-background.png`

移动端竖屏背景。

画面上半部分较暗、较干净，适合放 App 名称或 Logo；底部可以叠加登录按钮、启动按钮或简短说明。栏杆和花田保留了原图的视觉识别度，但构图更适配 9:16 竖屏。

推荐使用场景：

- App 启动页
- 移动端登录页
- Onboarding 第一屏
- 手机壁纸式产品视觉

## 前端使用建议

网页端建议优先使用 `2-web-hero-background.png`。它的文字安全区更明确，适合直接叠加白色或浅色文字。

移动端建议使用 `4-mobile-ui-background.png`。标题可以放在上方居中或偏上区域，按钮放在底部安全区。

如果页面需要更强的可读性，可以在背景上叠加一层暗色渐变遮罩：

```css
.hero {
  background:
    linear-gradient(90deg, rgba(7, 10, 12, 0.72), rgba(7, 10, 12, 0.18)),
    url("./2-web-hero-background.png") center / cover no-repeat;
}
```

移动端可以使用上下渐变遮罩：

```css
.mobile-hero {
  background:
    linear-gradient(180deg, rgba(5, 8, 10, 0.62), rgba(5, 8, 10, 0.12) 45%, rgba(5, 8, 10, 0.72)),
    url("./4-mobile-ui-background.png") center / cover no-repeat;
}
```

## UI 风格建议

搭配这组背景时，界面风格建议保持克制：

- 字体：使用清晰的无衬线字体，避免过度装饰。
- 颜色：主文字用白色或暖白色，辅助文字用低透明度白色。
- 按钮：使用简洁的实心按钮或细边框按钮。
- 面板：如果必须使用面板，建议用低透明度深色玻璃效果。
- 动效：适合轻微淡入、慢速背景缩放或视差，不适合强烈弹跳动画。

## 设计关键词

电影感、自然、隔阂感、低角度夕阳、花田、前景虚化、温暖高光、冷色阴影、沉浸式背景、克制 UI。

## 动态界面扩展

新增 `dynamic-ui.html`、`dynamic-ui.css`、`dynamic-ui.js`，将这组图进一步改成可运行的前端动态原型。

这版不再把图片只当作背景，而是拆成多个叠加层：

- 影像底层：使用 `2-web-hero-background.png` 和移动端的 `4-mobile-ui-background.png` 作为氛围基础。
- 动态自然层：通过 Canvas 绘制花粉/尘埃粒子，让画面有持续漂浮感。
- 光影层：使用缓慢变化的斜向光带模拟夕阳扫过画面的感觉。
- 云雾层：用低透明度的大面积流动层制造天空和空气的移动。
- 前景层：栏杆作为独立前景层，根据鼠标位置产生轻微视差。
- UI 层：导航、Hero 文案、动态信号面板、模式切换和底部信息条都叠在场景里。

交互模式包括：

- `Golden`：默认电影夕阳状态，节奏较慢。
- `Wind`：粒子数量和移动速度增加，画面更有风感。
- `Dusk`：整体降低亮度和饱和度，变成更冷、更安静的傍晚状态。

这个版本更适合继续发展成：

- 动态落地页
- 摄影/自然作品集首页
- App 官网视觉首屏
- 沉浸式登录页
- 带氛围动效的产品介绍页
