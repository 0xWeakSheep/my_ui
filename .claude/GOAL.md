# GOAL — 最终目标

> 本项目要达到的终极状态。

---

## 核心目标

建立一个**个人 UI 设计作品集网站**，同时作为**可复用的素材与组件库**。

---

## 目标分解

### 1. 对外展示

主站是一个可访问的 Vercel 网站，具备以下能力：

- **首页**：以网格或卡片形式展示所有 UI 素材。每个卡片显示缩略图、标题、分类标签和简短描述。
- **详情页**：点击素材卡片进入详情页，展示该素材的完整交互效果。
- **响应式**：主站在桌面端和移动端都有良好的浏览体验。
- **搜索/筛选**：支持按分类、标签或关键词搜索素材。

### 2. 素材库

每个素材是一个独立的、可运行的 UI 演示：

- **独立运行**：每个素材可以脱离主站，直接在浏览器中打开。
- **技术自由**：素材的实现技术不做限制。简单素材可以是 HTML/CSS/JS；复杂素材可以引入 Canvas、WebGL、Three.js、GSAP 等。
- **可复用**：素材中沉淀的组件、动效、颜色系统可以被抽取，用于未来的新作品或外部项目。
- **元数据完整**：每个素材都有 `meta.ts`（标题、分类、描述、版本、特性标签）和 `design.md`（设计思路、使用建议）。

### 3. 设计系统

跨所有素材共享的设计语言：

- **Design Tokens**：统一的颜色、字体、间距、圆角、阴影、动画时序。
- **组件规范**：基础 UI 组件（Button、Card、Badge 等）和预览组件（WorkCard、WorkGrid 等）有统一的实现规范。
- **玻璃效果**：跨作品共享的玻璃面板模式。

## 当前阶段目标

### 阶段 1：主站骨架（当前）

- 初始化 Next.js + Tailwind + TypeScript 项目。
- 搭建基础目录结构（`app/`、`components/`、`registry/`、`data/`、`styles/` 等）。
- 配置 Design Tokens（`styles/tokens.css` + `tailwind.config.ts`）。
- 实现首页：素材网格布局，每个素材显示为卡片。
- 实现详情页路由：`/works/[slug]`，点击卡片可进入。
- 详情页目前只需能正常进入，渲染方式后续确定。

### 阶段 2：素材迁移

- 将现有作品（`1/` 目录下的 Cinematic Field Interface）迁移到 `registry/01-field/`。
- 创建 `meta.ts` 和 `design.md`。
- 确保素材能独立运行。

### 阶段 3：渲染方式确定

- 根据素材复杂度确定详情页的渲染策略。
- 简单素材：iframe 嵌入或静态托管。
- 复杂素材：独立页面跳转或特殊处理。

### 阶段 4：扩展与优化

- 添加搜索、筛选、分类功能。
- 优化首页的视觉效果和交互。
- 添加新的 UI 素材。
- 持续沉淀可复用组件到 `components/ui/`。
